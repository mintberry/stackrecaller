CodeFish - A user study of fisheye visualization of source code
 Please take the time to read this introduction. It will be available as a reference manual at all times through the tab interface above.

When you have finished reading this introduction please click on task list 1 in the tab interface above.

Below is an overview of the various elements in the user interface
  • Focus area - The sightly yellowish area. No distortion of the code is performed here.
  • Context area - The area above and below the focus area. Empty lines, comments and preprocessor statements   are always filtered out. All other selection depends on the fisheye strategy.
  • Overview area - Creates a visual overview of the code structure and length.
  • Referenced lines - Lines containing variables or methods used within the focus area are highlighted with a light blue background color.
  • Selected line - The line below the mouse pointer is highlighted with blue.
  • Connectors - Each visible line in the code view are connected to its location in the overview.
Navigation
  • Keyboard - Up and down arrow, Page Up, Page Down, Home and End moves the focus area.
  • Mouse - Scroll wheel moves focus area up and down. Clicking in context view or overview centers the focus area at the clicked line. Click + drag in the overview moves the focus area accordingly.

Searching in the source code

  • Press 'CTRL + F' or click the 'Find' button in the main toolbar to show the find dialog
  • ESC closes the dialog

Fisheye strategies

  • Semantic - Lines in the context area are selected according to predefined semantic and syntactic rules. Rules include prefering block statements such as loops and conditions and closing brackets.
  • Dynamic - Regions in the context area are chosen from user interaction history. Regions which have been in focus for long are weighted heigher and regions visited half a minute ago are chosen preference of those visited five minutes ago.
